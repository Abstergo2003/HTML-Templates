CORS (Cross-Origin Resource Sharing) is a security feature implemented by web browsers that blocks web pages from making requests to a different domain than the one that served the web page. Since the chrome extension is not hosted on the same domain as the web page, it can be blocked by CORS policy.

There are a couple of ways to work around this issue:

  1. You can use a CORS proxy, which is a server that acts as a middleman between the extension and the website. The extension would make a request to the CORS proxy, and the proxy would then make the request to the website and return the response to the extension.

  2. You can use the chrome.webRequest API to modify the headers of the request and add the necessary headers to bypass CORS.

chrome.webRequest.onBeforeSendHeaders.addListener(
    function(details) {

        return {requestHeaders: details.requestHeaders};
    },
    {urls: ['<all_urls>']},
    ['blocking', 'requestHeaders']
);
  1. You can also serve your extension from a web server and specify the necessary headers to allow CORS.
